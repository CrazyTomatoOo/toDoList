T7: Task Form Validation Error Handling
========================================

Validation Error Cases:
-----------------------

1. Invalid Date Format
   - Triggered by: malformed date strings (not YYYY-MM-DD)
   - Handler: validateDateOnly() from src/shared/utils/dateValidator.ts
   - Error message: "{field} must be a valid YYYY-MM-DD date"
   - Displayed in: main form error div (data-testid="task-form-error")

2. Start Date After End Date
   - Triggered by: start_date > end_date when both present
   - Handler: validate() function in TaskForm.tsx
   - Error message: "Start date must be before or equal to end date"
   - Displayed in: DurationFields component (data-testid="task-form-duration-error")
   - Behavior: prevents form submission, shows inline error

3. Recurrence Without Base Date
   - Triggered by: recurrence selected but no due_date or start_date
   - Handler: validate() function in TaskForm.tsx
   - Error message: "Recurring tasks require a due date or start date"
   - Displayed in: form error div (data-testid="task-form-recurrence-error")
   - Behavior: prevents form submission, shows inline error
