## Badge Display Logic

### Recurrence Badge
- Shows: "Daily", "Weekly", "Monthly", or "Yearly"
- Condition: `task.recurrence !== null`
- Style: Accent color (blue) with light background

### Duration Badge
- Shows: "start_date → end_date" (formatted as "Jan 1 → Jan 7")
- Condition: `task.start_date !== null || task.end_date !== null`
- Handles partial dates with "…" placeholder
- Style: Secondary color (gray) with tertiary background

### Quadrant Badge
- Shows: "Q1", "Q2", "Q3", or "Q4"
- Condition: Always shown (defaults to Q4 if both flags are 0)
- Mapping:
  - Q1: urgent=1, important=1
  - Q2: urgent=0, important=1
  - Q3: urgent=1, important=0
  - Q4: urgent=0, important=0
- Style: Secondary color (gray) with tertiary background, semibold weight

## Design Decisions

1. **Quadrant badge always visible**: Unlike recurrence and duration, quadrant is always shown because every task has a quadrant classification (defaults to Q4).

2. **Helper function exported**: `getQuadrantLabel()` is exported for potential reuse in T10 (QuadrantBoard) and other components.

3. **Consistent styling**: All badges follow the existing priority badge pattern (small font, padding, border-radius) for visual consistency.

4. **Date formatting**: Reuses existing `formatDate()` helper for consistency with due_date display.

5. **Fallback behavior**: Invalid or missing data renders gracefully - quadrant defaults to Q4, partial dates show "…", missing recurrence hides the badge.
